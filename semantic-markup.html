Semantic Markup

semantic – relating to meaning

Always ask the question: "What purpose or function does that HTML element have?


Newer meaningful semantic elements

these replace the generic <div> element, that represents a general container, with a more specific container

Semanic Element Semantic Meaning
<section> Represents a standalone section of content on the page. Similar to sub or sub-sub headings.
<article> Represents a self-contained composition in a document, page, site; which is intended to be independently distributed or reusable. These are accompanied by a heading explaining the content.
<nav> The part of the page whose purpose is to provide navigational links, either within the current page or to other pages on the website; or other external resources.
<main> Should be unique to the document. Just the main content of the page, that is repeated across a set of documents or document sections. The function is strictly informative.
<header> displays introductory content, typically a group of introductory or navigational aids. A header may contain some heading elements but also a logo, a search form, an author name, or other elements.
<footer> A footer typically contains information about the author of the section, copyright data, trademarks, or links to related documents.
<aside> Is a portion of a document whose content is only indirectly related to the main content of the document. Asides are frequently presented as sidebars or call-out boxes.
<summary> Should be unique to the document. Content that is repeated across a set of documents or document sections. The function is strictly informative.
<details> Should be unique to the document. Content that is repeated across a set of documents or document sections. The function is strictly informative.

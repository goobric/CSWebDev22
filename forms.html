Forms

The <form> element represents a document section containing interactive controls for submitting information.

The method attribute specifies which HTTP method should be used.

The action attribute specifies where the form data should be sent.

the form itself is a container, with 'fields' that hold input values from the user. Illustrated by the image.

the most common element within a form is the <input> element.

there are over 20 possible types of 'input' ranging from checkboxes to date pickers.

the type attribute is where the magic happens. Changing the type alters the behaviour and appearance of the input

this is an input example from MDN.

Important

the <label> element is extremely important, with forms and input types. The label links together the value of the input from the user with the input type. The text from the label is associated with the input type. Selecting the text associated with the input, activates the input element.

the <label> has two corresponding attributes: for and id these MUST contain the same value, which is unique for that label and input.

inspecting the above form, using the Dev Tools, shows that each label has a unique for and id value.

Buttons

when a <button> element is placed inside the form element, the default action is to complete the <form action="">

when a <button> element is placed outside of the form element, there is no default behaviour.
